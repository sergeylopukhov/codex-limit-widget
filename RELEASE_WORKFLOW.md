# Выпуск версий

## Правила

- `main` — стабильная версия для пользователей.
- `beta` — тестовая ветка для новой работы.
- Каждая новая функция или исправление сначала попадает в `beta`.
- Тестовые версии имеют теги вида `v1.1.11-beta.1`. GitHub отмечает их как prerelease, поэтому они не предлагаются как обновление стабильной версии.
- Стабильный релиз выпускается только после прямой команды Сергея: «выпускай релиз».
- Публикация не устанавливает и не открывает приложение на локальном Mac. Локальная установка выполняется только по отдельной команде, например «установи локально».

## Выпустить beta

1. Оставить изменения в `beta` и проверить сборку.
2. Обновить версию везде: Xcode-проекте, обоих `Info.plist`, `scripts/make-release.command` и `CHANGELOG.md`.
3. Закоммитить и отправить `beta` на GitHub.
4. Создать и отправить тег вида `vX.Y.Z-beta.N`.
5. GitHub Actions автоматически соберёт DMG и ZIP и создаст GitHub prerelease.

Пример: `v1.1.11-beta.1`.

## Выпустить стабильную версию

Делать только после проверки beta.

1. Взять проверенный код из `beta`.
2. Обновить версию с beta до `X.Y.Z` везде и добавить финальную запись в changelog.
3. Применить результат в `main` одним чистым коммитом `Release X.Y.Z`.
4. Отправить `main`, создать и отправить стабильный тег `vX.Y.Z`.
5. GitHub Actions автоматически соберёт DMG и ZIP и создаст стабильный GitHub Release.
6. Не устанавливать приложение локально без отдельной просьбы Сергея.

Так `main` остаётся чистой стабильной веткой, а `beta` — местом для дальнейших экспериментов.
